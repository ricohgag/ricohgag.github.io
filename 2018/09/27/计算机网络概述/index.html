计算机网络概述

互联网
  • 互联网是网络的网络，能将各种不同的网络连接起来。
ISP
  • ISP是互联网提供商，向用户综合提供互联网接入业务、信息业务和增值业务的电信运营商。
  • 互联网是多层ISP结构：
    • 第一层为ISP的服务面积最大(一般能覆盖国家范围)，并且拥有高速主干网。
    • 第二层ISP和大公司都是第一层ISP的用户
    • 第三层ISP（本地ISP），是第二层ISP的用户，且只拥有本地范围的网络，一般的校园网或者企业网以及拨号上网的用户都是第三层ISP的用户。
主机之间的通信方式
  • 客户-服务器（C/S）：客户是服务的请求方，服务器是服务的提供方。
  • 对等（P2P)：不区分客户和服务器。
电路交换与分组交换
  1. 电路交换
     电路交换用于电话通信系统，两个用户通信之前需要建立一条专用的物理链路，在通信过程中会占用该链路。因为一直占用传输线路，因此电路交换对线路的利用率很低，往往不到10%。
  2. 分组交换
     每个分组都有首部和尾部，包含了原地址和目的地址等控制信息，在同一个传输线路上同时传输多个分组互相不会影响。所以说分组不需要占用线路。
     分组交换还使用了存储转发过程，收到分组后先存储下来，把统一目的的分组一起转发到下一个目的地。
时延

分组交换时，分组从一台主机(源)，通过一系列路由器传输，到达另一台主机(目的地)时。该分组在沿途的每个节点都经过了不同类型的时延。

总时延 = 传输时延 + 传播时延 + 处理时延 + 排队时延。

  1. 传输时延
     数据帧从结点(主机)进入到信道(传输线路)所需要的时间。
    s(时间) = l(数据帧长度)/v(传输速率)

  2. 传播时延
     分组在信道中传播所需要花费的时间，电磁波传播速度接近光速。
    s(时间) = l(信道长度)/v（电磁波速度）

  3. 处理时延
     主机或路由器接收到分组时进行处理分析需要的时间，如分析首部、从分组中提取数据、进行差错检验或查找适当的路由等。

  4. 排队时延
     分组在路由器输入队列和输出队列等待的时间，取决于当前网络的通信量。

互联网体系结构
  1. 五层协议
    • 应用层：为特定应用程序提供数据传输服务，协议如HTTP、DNS等，数据单位为报文。
    • 传输层：为柱击中的进程提供通用数据传输服务。通用的数据传输服务可以支持不断增多的应用层协议。传输层包括两种协议：传输层控制协议TCP和用户数据报协议UDP。
      • TCP：主要提供完整性服务，提供面向连接、可靠的数据传输服务，数据单位为报文段。
      • UDP：主要提供及时性服务，提供无连接、尽最大努力的数据传输服务，数据单位为用户数据报。
    • 网络层：为主机之间提供数据传输服务。网络层把传输层传递下来的报文段或用户数据报封装成分组。
    • 数据链路层：链路层协议为同一链路下的主机的提供数据传输服务，数据链路层把网络层传下的数组封装成帧。
    • 物理层：考虑的是怎样在传输媒体(信道)上传输数据比特流，物理层的作用是尽可能的屏蔽传输媒体和通信手段的差异，使数据链路层感觉不到差异。
  2. OSI

     OSI将应用层分为应用层、表示层和会话层。

    • 表示层：数据压缩、加密以及数据描述，这使得应用程序不用关系在各台柱击中数据内部格式不同的问题。

    • 会话层：建立及管理会话。

      五层协议没有表示层和会话层，这些功能留给应用程序开发者处理。

  3. TCP/IP体系结构

     TCP/IP体系结构只有四层：应用层、运输层、网际层和网络接口层。此结构不严格遵守OSI分层概念，应用层可能会直接使用IP层或网络接口层。
     TCP/IP协议簇是一种沙漏形状，中间小两边大，IP协议在其中占有重要的地位。

  1. 数据在各层之间的传递
     在向下的过程中,需要添加下层协议所需要的首部或尾部，再向上的过程不断拆开首部和尾部。
     路由器只有下三层协议，因为路由器位于网络核心中，不需要为进程或应用程序提供服务，因此也不需要传输层和应用层。
