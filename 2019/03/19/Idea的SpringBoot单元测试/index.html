Idea的SpringBoot单元测试

发表于 2019-03-19 | 分类于 测试

SpringBoot单元测试

* 环境配置
  • 安装Idea 单元测试插件
    1.进入File-Settings-Plugins，点击Browse Repositories
    2.搜索 JunitGenerator 使用v2.0版本，下载安装
    3.重启之后在Settings-Other Settings里找到 JunitGenerator,按图中选择。 output Path填入: \${SOURCEPATH}/../test/java/${PACKAGE}/${FILENAME}




* 编写测试方法

1.在需要测试的业务方法类里 ctrl+shift+T 选择需要添加单元测试的方法。


2.在测试类上加上@RunWith，@SpringBootTest注解，启动时先加载容器


3.@Autowired注入测试目标类，（Ioc容器在测试加载时已经准备完毕，所以可以直接注入）
4.在测试方法里针对测试目标方法所有的情况编写测试用例。
5.在chooceByNum方法里num作为输入有3种情况，测试用例需要把三种情况都考虑进来。当代码逻辑分支更改时，需要跟着更改单元测试的测试用例，保证测试用例能够全部能够绿色(运行成功)。
6.Assert是用来比较输出是否达到理想的效果，如果不一致单元测试会报错。

* 单元测试的意义
1.程序员能在开发环节提前发现错误，而不是在测试环节甚至生产环境中得知这个消息，减少了犯错的成本和机会。
2.在测试用例考虑周全的情况下，当系统重构或更改代码后，单元测试能保证更改的代码的安全性
